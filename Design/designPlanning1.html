PLANNING

IxD Process:


This involves: Goal Alignment, Information Structure - content mapping, User flow - screens, behaviour and decisions, Interface elements and Design - Style guide & wireframes

Implementation

Goal Alignment:


My users would hopefully be potential collaborators or employers. What those users would be able to do is view the art that I create, and be able to contact me regarding that art if they so please. The content on the site would be artworks and writing pieces that I have completed so far.

Information structure


I have created a mind map and a flow diagram to display how the different pages of my website are linked and how they flow together. This displays the page dependancies.

User Flow


Users will have to press a button (with the page's name on it) to access each page. They will never find themself on a page by accident. Clicking on a page's corresponding button would take it to the page itself. Each sepaate component would have its own page, for example, clicking on blogs would take users to the blog homepage (which is it's own page). From there, users would click on the blog that they want to read, which would take them to that specific blog's page.

Interface elements


Each page should have things to interact with, so that its not just read or viewing. An idea that I've been having is to create a character that hides somewhere on each page, and users would find them by exploring the page. Once they have found them, they can click on them and then there would be a small celebratory segment (like confetti showering the screen or something like that) to denote that they've found them. It would be like Where's Wally or Find Mii from Wii Play.

Design - Style guides and wire frames


For this section, I have created visual mockups based on my wireframes to display what the page would look like. This includes the colour palette, fonts used and button designs. Each section has it's own colour scheme, with secondary colours that are consistent throughout the site. Design philosphy-wise, I only want each page to only contain the most essential interactable options for users. I want the page to minimalistic in aesthetic and options to not visually overload the user. This will make the content I do have on the page more meaningful, as it is what is there for them to see. Nothing more, nothing less.

Homepage

The central hub of the entire site. This is where users will be able to access all of the different sections that the site has to offer. I want this to be as transparent as possible, meaning that it shouldn't be cluttered with anything unnecessary to the site like personal information.

Blog Homepage

The space to access blogs. This should mimic the design sense of the homepage, as it is like a mini homepage, just for the blogs. This space is updated everytime a blog is added to the site, as users wouldn't be able to access that blog without it.

Individual Blog Page

What each blog page will look like. In a sense, they mimic the aesthetics of messaging applications like iMessage or Instagram Direct Messages. It's like this because I wanted my site to feel like I was talking directly to users.

Essay Homepage

The space to access essays. This should mimic the design sense of the homepage, as it is like a mini homepage, just for the essays (much like the blogs set up). This space is updated everytime an essay is added to the site, as users wouldn't be able to access that essay without it.

Individual Essay Page

What each essay page will look like. Much like the blogs, they mimic the aesthetics of messaging applications like iMessage or Instagram Direct Messages. Again, it's like this because I wanted my site to feel like I was talking directly to users.

Inspirational Site:


Nintendo eShop

What I like:

  • Navigation is smooth, everything you need to access is always close by.
  • Strong branding comes through with the colour palette.
  • The way updates (like new games or sales) are displayed makes it feel like the site always has something new to offer.

What I don't like:

  • It often feels a bit too busy.
  • The text is a bit too small.
  • If there are a lot of updates, it can feel like information overload.
